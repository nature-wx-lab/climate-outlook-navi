# 気候ものさしナビ

Nature Wx Lab第4弾の公開ツールです。全国陸域387,717個の1kmメッシュで独自算出した8つの気候要素と、気象庁の1か月予報・3か月予報が示す地域別3階級確率を、同じ地点で読み比べます。

- 日本語名: 気候ものさしナビ
- 英語名: Climate Outlook Navi
- 公開URL: `https://nature-wx-lab.github.io/climate-outlook-navi/`

## 実装範囲

- 要素: 平均気温、日最高気温の平均、日最低気温の平均、降水量、日照時間、最深積雪、降雪量、全天日射量
- 気候期間: `1991_2020` / `1996_2025`
- 区分: 月別1〜12と年値13。全天日射量だけは月別1〜12で、年値を合成しない
- 表示: 絶対値 / `30年平均値の更新差（1996–2025 − 1991–2020）`
- 地点参照: 各要素の全387,717メッシュを176個の1次メッシュ別バイナリへ分割。合計79,869,702値を元データと同じ0.01単位で保持
- 全国表示: 8要素合計309枚のWebP描画縮約。地点値は縮約せず、要素別バイナリから参照
- 季節予報: 平均気温↔気温、降水量↔降水量、日照時間↔日照時間、降雪量↔降雪量を直接対応。対応商品がない要素は無理に重ねない
- 予報商品: P1M / P3Mの公式class15s境界へ展開。日照時間はP1Mだけ、降雪量は季節限定。最大確率が同率の場合は灰色の「同率首位」と明示
- 操作: PCはカーソル位置の地点値・予報地域を右パネルへ最大10回/秒でプレビュー。クリック／タップで1km地点を固定し、2期間の1〜12月グラフを表示。固定解除、別地点選択、現在地、表示状態URL、PNG保存、ベース地図、透明度に対応
- 初期画角: URLに画角指定がない時は中心`37.2°N, 137.2°E`・zoom 5で日本列島を大きく表示。「全国へ戻る」も、離島を含む広すぎるbounds fittingではなく同じ画角へ戻す。共有URLに明示された画角はそのまま復元する
- 地図境界: デフォルトは都道府県境だけを実線表示。季節予報のclass15地域は色面・tooltip・クリック判定を維持し、県内を区切る地域境界線は描かない

最深積雪の年値は、観測地点ごとに12個の月別平均最深積雪の最大を作り、その地点値を月別面とは別に1km内挿した「年値（最大月平均）」です。寒候年ごとの年最深積雪の平均でも、同じ1kmメッシュの月別面12枚の最大でもありません。

## データ生成

共有気象基盤DBは読み取り専用です。入力先をコードへ固定せず、環境変数か引数で渡します。

```bash
export NATUREWXLAB_DB_ROOT="<shared-climate-db-root>"
python3 scripts/build_climate_data.py \
  --prefectures <prefecture-boundary.geojson>
python3 scripts/copy_leaflet_vendor.py \
  --source <leaflet-1.9.4-directory>
```

生成物は `data/climate/` の派生ファイルだけです。`catalog.json`が8要素のmanifestと全公開資産のbyte数・SHA-256を列挙し、201の既存chunk v1と追加7要素の自己記述chunk v2を同時に管理します。共有DBのgzip CSVや絶対パスは公開物へ含めません。

builderは全入力のschema・行数・mesh順・0.01単位の量子化誤差・固定地点値・`日最低気温の平均 ≤ 平均気温 ≤ 日最高気温の平均`を検査します。出力は隔離したstagingへ作り、許可ファイル一覧、全checksum、binary再読込が成功した場合だけ一式をatomicに切り替えます。同じ入力・同じ生成設定での再実行はno-opです。

季節予報は次で更新します。

```bash
python3 scripts/update_season_forecast.py
python3 scripts/verify_contract.py
python3 scripts/privacy_gate.py
```

更新処理はP1M/P3M、4予報要素、class15s、area hierarchyを取得し、schema・確率合計・地域展開・境界件数を検査します。未対応商品と季節外の未発表を区別し、降雪量は公式に値がある地域だけを保持します。正常なcandidateだけをversioned datasetとして保存し、最後に `data/season/manifest.json` をatomicに差し替えます。取得や検査に失敗した場合は、前回正常datasetへの参照を保持します。

`datasets/` は現行manifestが参照するdatasetと直前正常版の計2世代を既定で保持します。promotion成功後だけ古い世代を削除するため、取得・検査失敗時のlast-known-goodは維持され、未参照datasetも無制限には蓄積しません。保持数は `--keep-datasets 1..5` で変更できます。

画面側でも対象期間終了と保守的な発表経過日数（P1M 9日、P3M 45日）を再判定し、古いデータを`利用可能`のまま固定表示しません。P3Mの本番更新スケジュールは、今後、気象庁の年次発表カレンダー連動へ更新する余地があります。

## ローカル起動

リポジトリ直下をHTTP配信します。

```bash
python3 -m http.server 8765
```

ブラウザで `http://127.0.0.1:8765/` を開きます。`file://` ではバイナリとJSONを読み込めません。

## 自動更新と公開

- `privacy-gate.yml`: 現行ファイル、到達可能なGit履歴、commit message、author/committer identityを検査します。
- `update-and-deploy.yml`: 毎日15:05 JSTごろに公式P1M/P3Mを確認し、内容が変わった場合だけGitHub Actions botで更新します。同じrunで契約検査済みのallowlist成果物だけをPagesへ配信し、公開後に全配信ファイルのchecksumを再確認します。

公開用Git identityは `nature-wx-lab` のGitHub noreply identityとGitHub Actions botだけを許可します。Web UI編集は使用しません。

## 表示契約

- `1991–2020年平均（独自算出・気象庁平年期間）`を「気象庁の平年値」と呼びません。
- 全国ラスターは初期表示を軽くするための描画縮約です。地点値は全387,717メッシュを保持する分割バイナリから読みます。
- 左設定欄は上部タイトルと重複する見出しを置かず、最初の要素選択から表示します。
- 天気分布予報プラスと同様、通常表示の行政境界は都道府県境だけです。季節予報の地域差は色面で表し、地域ポリゴンの点線は重ねません。
- カーソル追従はfine pointer端末だけで有効です。クリック後は地点を固定してカーソル移動で上書きせず、「固定解除」でプレビューへ戻します。タッチ端末はタップで固定地点の詳細を開きます。
- 固定地点のグラフは2つの30年平均の月別値を比較するもので、季節予報の時系列ではありません。年値は月別グラフから再計算せず、最深積雪の年値は月別12点の最大値でもありません。
- 季節予報は地域単位の確率です。1kmへ補間せず、確率から絶対気温や偏差量を作りません。
- 日最高・日最低気温の平均を、季節予報の平均気温と同一要素として扱いません。最深積雪と降雪量も区別します。
- 全天日射量は `MJ/㎡/日` の日平均で、月合計や年値として表示しません。
- 日最低気温の平均は監査時LOOCV 1.06℃、雪要素は観測地点が疎な地域の独自内挿に注意が必要です。
- `P1M.json` / `P3M.json` は気象庁Web地図の運用JSONであり、schema変更に備えて更新時に毎回検証します。
